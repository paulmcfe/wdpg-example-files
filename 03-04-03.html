The basic form of any URL looks like this: 

How://Who/Where/What

Let’s see what all this means:

  • How — The first part of the URL specifies how the data is going to be transferred. This is called the protocol and, luckily, mere mortals like you and I don’t need to concern ourselves with the guts of this stuff. All you need to know is which protocol each resource uses, which is straightforward. For example, the web uses something called HTTP. The “how” part of the URL is the protocol, followed by a colon (:) and two slashes (//). So, a web page URL always starts like this: http://.
  • Who — Calling the next part the “who” of the URL is, I admit, a bit of a misnomer because there’s no person involved. Instead, it’s the name of the server where the resource is located — this is called the host name. For example, the Web Design Playground is located on a host named webdesignplayground.io. To continue the URL, you tack this “who” part onto the end of the “how” part: http://webdesignplayground.io.
  • Where — The next part of the address specifies where the resource is located on the host. This generally means the directory in which the resource is stored; the directory might be something like /www or /public/html. The Web Design Playground is in the root directory, which is /. So now you append the directory onto the URL and then add another slash on the end, for good measure: /.
  • What — Almost there. The “what” part is the name of the file you want to reference. For a web page, you use the name of the document that contains the HTML code and text. The file containing the Wrb Design Playground is called index.html, so here’s the full URL: http://webdesignplayground.io/index.html.
